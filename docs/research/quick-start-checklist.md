# Research Integration Quick-Start Checklist

Use this once research is actively shaping Convergence implementation.

## Before Starting a Task

- [ ] I know which architecture doc or prototype this task affects.
- [ ] I checked `master-index.md` for relevant research.
- [ ] I read the primary translation memo for this area.
- [ ] I noted any required prototype or validation work.

## During Implementation

- [ ] I am following research recommendations or documenting a deliberate deviation.
- [ ] I am referencing the research basis where code behavior depends on it.
- [ ] I am noting any research gaps I discover.

## Before Submitting

- [ ] I created an implementation decision record if the task introduced material design choices.
- [ ] I added tests or validation steps that reflect the research basis.
- [ ] I verified no required prototype is still silently pending.

## When Research Is Missing

- [ ] I documented the gap.
- [ ] I made the provisional decision explicit.
- [ ] I added the gap to `gaps-found-during-implementation.md`.
